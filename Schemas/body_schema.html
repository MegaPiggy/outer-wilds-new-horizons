Celestial Body Schema

Type: object

A planet or body to generate

No Additional Properties

Root $schema
Type: string

The schema used to validate your config.

Root name
Type: string

Unique name of your planet

Root starSystem
Type: string
Default: "SolarSystem"

Unique star system containing your planet

Root destroy
Type: boolean
Default: false

true if you want to delete this planet

Root removeChildren
Type: array of string
Default: []

A list of paths to child GameObjects to destroy on this planet

Each item of this array must be:

Root removeChildrenremoveChildren items
Type: string

Root canShowOnTitle
Type: boolean
Default: true

Should this planet ever be shown on the title screen

Root isQuantumState
Type: boolean
Default: false

Does this config describe a quantum state of a custom planet defined in another file?

Root Base
Type: object

No Additional Properties

Root BasehasMapMarker
Type: boolean
Default: false

If the body should have a marker on the map screen.

Root BasehasAmbientLight
Type: boolean
Default: false

If the dark side of the body should have some slight amount of light

Root BasesurfaceGravity
Type: number
Default: 12

The acceleration due to gravity felt as the surfaceSize. Timber Hearth has 12 for reference

Root BasegravityFallOff
Type: enum (of string)
Default: "linear"

How gravity falls off with distance. Most planets use linear but the sun and some moons use inverseSquared.

Must be one of:

  • "linear"
  • "inverseSquared"

Root BasesurfaceSize
Type: number
Default: 100

A scale height used for a number of things. Should be the approximate radius of the body.

Root BasesphereOfInfluence
Type: number
Default: 0

An override for the radius of the planet's gravitational sphere of influence. Optional

Root BasegroundSize
Type: number
Default: 0

Radius of a simple sphere used as the ground for the planet. If you want to use more complex terrain, leave this as 0.

Root BasehasCometTail
Type: boolean
Default: false

If you want the body to have a tail like the Interloper.

Root BasecometTailRotation
Type: object

If it has a comet tail, it'll be oriented according to these Euler angles.

No Additional Properties

Root BasecometTailRotationx
Type: number
Default: 0

Root BasecometTailRotationy
Type: number
Default: 0

Root BasecometTailRotationz
Type: number
Default: 0

Root BasehasReferenceFrame
Type: boolean
Default: true

Allows the object to be targeted on the map.

Root BasecenterOfSolarSystem
Type: boolean
Default: false

Set this to true if you are replacing the sun with a different body. Only one object in a star system should ever have this set to true.

Root BaseshowMinimap
Type: boolean
Default: true

Do we show the minimap when walking around this planet?

Root BasecloakRadius
Type: number
Default: 0

Radius of the cloaking field around the planet. It's a bit finicky so experiment with different values. If you don't want a cloak, leave this as 0.

Root BaseinvulnerableToSun
Type: boolean
Default: false

Can this planet survive entering a star?

Root Orbit
Type: object

No Additional Properties

Root OrbitsemiMajorAxis
Type: integer
Default: 5000

The semi-major axis of the ellipse that is the body's orbit. For a circular orbit this is the radius.

Value must be greater or equal to 0

Root Orbitinclination
Type: number
Default: 0

The angle (in degrees) between the body's orbit and the plane of the star system

Value must be greater or equal to 0 and lesser or equal to 360

Root OrbitprimaryBody
Type: string
Default: "Sun"

The name of the body this one will orbit around

Root OrbitisMoon
Type: boolean

Is this the moon of a planet? Used for determining when its name is shown on the map.

Root OrbitlongitudeOfAscendingNode
Type: number
Default: 0

An angle (in degrees) defining the point where the orbit of the body rises above the orbital plane if it has nonzero inclination.

Value must be greater or equal to 0 and lesser or equal to 360

Root Orbiteccentricity
Type: number
Default: 0

At 0 the orbit is a circle. The closer to 1 it is, the more oval-shaped the orbit is.

Value must be greater or equal to 0 and lesser or equal to 0.99999

Root OrbitargumentOfPeriapsis
Type: number
Default: 0

An angle (in degrees) defining the location of the periapsis (the closest distance to it's primary body) if it has nonzero eccentricity.

Value must be greater or equal to 0 and lesser or equal to 360

Root OrbittrueAnomaly
Type: number
Default: 0

Where the planet should start off in its orbit in terms of the central angle.

Value must be greater or equal to 0 and lesser or equal to 360

Root OrbitaxialTilt
Type: number
Default: 0

The angle between the normal to the orbital plane and its axis of rotation.

Value must be greater or equal to 0 and lesser or equal to 360

Root OrbitsiderealPeriod
Type: number
Default: 0

Rotation period in minutes.

Root OrbitisTidallyLocked
Type: boolean
Default: false

Should the body always have one side facing its primary?

Root OrbitalignmentAxis
Type: object

If it is tidally locked, this direction will face towards the primary. Ex: Interloper uses 0, -1, 0. Most planets will want something like -1, 0, 0.

No Additional Properties

Root OrbitalignmentAxisx
Type: number
Default: 0

Root OrbitalignmentAxisy
Type: number
Default: 0

Root OrbitalignmentAxisz
Type: number
Default: 0

Root OrbitshowOrbitLine
Type: boolean
Default: false

Referring to the orbit line in the map screen.

Root OrbitdottedOrbitLine
Type: boolean
Default: false

Root OrbitisStatic
Type: boolean
Default: false

Is the body meant to stay in one place without moving?

Root Orbittint
Type: object

Colour of the orbit-line in the map view.

No Additional Properties

Root OrbittintR
Type: integer
Default: 0

Value must be greater or equal to 0

Value must be greater or equal to 0
